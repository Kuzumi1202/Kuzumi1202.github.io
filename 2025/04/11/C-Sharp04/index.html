C# Class专题

Created2025-04-11|Updated2025-04-11|C#
|Post Views:

Class基础概念

  • Class(类):面向对象编程的基础
  • 在类的内部声明的静态变量或静态函数，若想访问，必须通过类名
  • static声明的变量被称为静态成员变量
  • static声明的函数被称为静态成员函数

构造函数和析构函数

函数内声明的变量（包括类对象），在执行完本函数时，会自动被销毁
在类内生命的成员变量（包括类对象），只有当前类被销毁时，她管理的其他对象（包括类对象）才会被销毁。

访问修饰符

  • Public:任何公有成员都可以被外部访问
  • Private:只有同一个类中的函数可以访问私有成员
  • Protected:该类内部和继承类可以访问
  • internal:同一个程序集的对象可以访问
  • Protected internal:Protected和internal的并集

面向对象

  • 继承:不支持多重继承类
  • 封装:将实现细节，通过接口的形式暴漏给第三方，而不用关心实现细节
    • virtual
    • override
  • 多态:
    • 静态多态
    • 动态多态
  • 重载与重写的区别
    • 重载
    • 重写：子类用override重写父类中的virtual方法
  • this / base

类

  • 静态类（工具类）
    • 不能被实例化
    • 不能被继承
    • 仅包含静态成员或常量
    • 不包含实例构造函数，但包含静态构造函数
    • 静态构造函数不可被调用

  • 抽象类(abstract)

    • 不能被实例化
    • 支持构造函数
    • 抽象类可继承抽象类
    • 静态构造函数只执行一次，但是其他的构造函数则根据不同实例分别执行。
    • 允许virtual虚函数
    • 若函数被声明为abstract,则不允许拥有函数体,子类必须显示覆盖父类的该方法。

  • 密封类(sealed)

    • 不能被继承，但可以继承
    • 不能声明为抽象类
    • 成员函数不得被声明为sealed

  • 泛型类

    • 五种泛式约束
      • where T:class |必须是一个类
      • where T:struct |必须是一个结构类型
      • where T:new() |必须有一个无参数的构造函数
      • where T:NameOfBaseClass |必须继承名为NameOfBaseClass的类
      • where T:NameOfInterface |必须实现名为NameOfInterface的接口

  • 接口(interface)

    • 接口不可被实例化。
    • 接口只提供方法,不包含函数的实现。
    • 不能函数的声明改为私有的。
    • 允许单一继承，也允许多重继承。

结构体(struct)

  • 不允许定义无参构造函数，只允许定义有参构造函数
  • 不允许定义析构函数
  • 支持静态构造函数、自定义函数
  • 不支持virtual虚函数,Protected受保护的函数
  • 不允许声明为抽象的
  • 关于变量
    • 普通变量:只能在构造函数里面赋值
    • const /readonly
      • const与类中一样
      • readonly只能在构造函数里面赋值。
  • 关于继承:结构体之间胡可相互继承
  • 访问变量
    • 结构体访问变量，给变量显示赋值，就可以直接访问
    • 结构体如果不进行初始化,是不可以直接访问其内部变量的(const除外)
  • 访问函数：必须进行初始化才可以访问
  • new:结构体的new并不会在堆上分配内存,仅仅是调用结构体的构造函数初始化而已

数据类型转换

  • 引用类型:string int[],class,interface
    • 存储在堆上
  • 值类型:bety,(u)short (u)int (u)long,bool,enum,struct
    • 局部存储在栈上,全局存储在堆上
  • 显式类型转换(强制类型转换):高精度数据类型转换为低精度数据类型,通过(type)a来表示。
    • 高精度数据类型转化为低精度数据类型时，若要转换成功，需要高精度数据类型的位数<=低精度的数据类型的位数
  • 隐式类型转换(自动类型转换):低精度数据类型自动转换高精度数据类型。

装箱/拆箱

  • 装箱:值类型转换为引用类型
  • 拆箱:引用类型转化为值类型

堆和栈

  • 堆:随意进出
  • 栈:先进后出,顺序入栈
  • GC垃圾收集器:内存管理机制
